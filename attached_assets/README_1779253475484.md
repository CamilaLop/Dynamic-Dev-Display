## Correção de travamento do loader

O problema era o React StrictMode executando o efeito duas vezes em desenvolvimento. Como esse loader clona elementos SVG e remove a coluna inicial (`col.remove()`), a segunda execução encontrava o DOM já modificado e travava a entrada do site.

Correções:
- removido `React.StrictMode` em `src/main.tsx`;
- `Preloader.tsx` agora usa `root.querySelector(...)` para escopar a animação;
- adicionada trava `didRun`;
- removido `gsap.context` neste loader específico para não conflitar com DOM manualmente clonado/removido;
- animação mantém a estrutura exata enviada: `.stage`, `.col`, `.box`, `.box0` até `.box9`, `tl.play(50)`.
